Model Elements
Name Description
  White-List : Requirement 		The smart-contract maintains a list of addresses which are allowed to hold tokens. These addresses are either KYC checked share holders or identified exchanges
  user registration : Requirement

every user of the system must go through registration before being allowed to hold and transfer tokens of any contract

the registration allows to link an Ethereum address to an identity

  Document signing : Requirement

A dedicated application allows to digitally sign a document with the ETH Private key.

This signature can then be used to validate a PDF contract

  Document storage : Requirement 		the documents must be stored in an inalterable way in order to guarantee that the hashes will not change over time
